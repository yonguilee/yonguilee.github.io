티스토리로 간간히 블로깅을 해오다가 코드블럭이 맘에 안들어서 다른 방법들을 찾아보다가 Github Pages로 블로그 만들기에 도전해보게 되었다.



Github Pages 생성을 위한 저장소 생성

github에서 블로그용 프로젝트를 생성한다.

  • 저장소 명은 ‘~.github.io’ 로 생성하기
  • 로컬에 설치


Hexo 설치

blog/ 하위 _config.yml에서 설정된 부분들을 개인에 맞게 커스텀 할 수 있다.

설정 파일 : _config.yml

_config.yml는 기본 설정 파일이다.
각 항목에 대한 자세한 설명은 Hexo Document > Configuration에서 확인 가능하다.

# Site

여기서 language는 테마의 기본 언어를 설정해주는 것으로, 아마.. 이 글을 읽는 사람은 모두 ko 라고 적겠지만, 다른 언어를 선택하고자 한다면 themes/languages에 있는 언어명 중에서 찾아 동일하게 적어주면 된다.

# Deployment

명령어

기본 페이지 생성

원하는 레이아웃(blog/scaffolds)으로 문서 source 파일(source/)을 생성한다.
디폴트 레이아웃은 post.md 이다.

로컬 빌드

로컬에서 돌려보면 4000번 포트로 확인 가능하다.

배포위한 페이지 생성

원하는 레이아웃(blog/scaffolds)으로 문서 source 파일(source/)을 생성한다.
디폴트 레이아웃은 post.md 이다.

배포

간혹, 수정한 내용이 반영안되는 경우가 있는데 아래 명령어를 수행 후 다시 generate 후 deploy 하면 된다.

테마 설치

Hexo를 설치하면 기본테마는 landscape 이다.
이 외 테마들은 Hexo 테마에서 확인할 수 있고 원하는 테마를 다운 받으면 된다.
나는 icarus를 적용할 것이다.
보통 테마 데모 화면을 보면 설치방법, 기능들에 대한 설명을 확인할 수 있다.

나는 icarus 데모 사이트의 Getting Started with Icarus를 참고 했다.

테마 저장소 git clone

테마를 clone하게 되면 themes/ 하위에 icarus 폴더가 생성된다.

테마를 설치하면 icarus폴더 내에도 _config.yml 가 생성된다.

같은 설정파일이지만, 설정할 수 있는 부분이 다르니 잘 찾아서 활용하자.

blog/_config.yml : 블로그 설정 파일

blog/themes/icarus/_config.yml : 테마 관련 설정 파일

blog/_config.yml 에서 테마 변경하기

파일을 저장하고, hexo server를 다시 돌려본다. 그럼 잘 절용되는 사람도 있을 것이고, 아닌 사람도 있을 것이다.
나같은 경우는 다음과 같은 메세지가 떴다.

cheerio가 설치 안되어 있다는 말이니, 설치해 주면 된다. 그럼 icarus 가 적용된 화면이 빌드된다.
