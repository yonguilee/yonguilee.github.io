Vue로 UI 개발하기 - 좋아요 컴포넌트(튜토리얼2)

‘좋아요 컴포넌트’는 자주 볼 수 있는 UI 중 하나이다. 이번 글에서는 앞에서 정리한 기능 구현 목록 중 썸네일 0-3개 노출, 썸네일 3개 이상일 경우 ‘외 N명’ 노출를 단계별로 살펴볼 것이다.

썸네일은 0 ~ 3개까지 노출

‘좋아요’를 누른 사용자 정보를 받아와서 보여주는 부분. 데이터를 어떻게 받아올지 모르겠지만, 더미 데이터를 만들어서 Vue 문법에 맞게 바꿔보겠다.(미리 협의가 된다면 동일한 데이터 객체를 만들어 테스트하면 된다.)
static한 화면을 먼저 만들고, css를 추가한다.


더미 데이터 만들기

userList라는 배열을 생성한다. 최대 3개 썸네일이 생성되도록 하는 것을 확인하기 위해 4개의 사용자 객체를 생성했습니다. 데이터에서 이미지 경로를 담을 때에는 require()로 묶어준다.


HTML에 데이터 연결하기

v-bind
HTML 태그에 추가할 수 있는 ‘v-‘로 시작하는 Vue 전용 속성.
v-bind는 html 속성에 Vue의 데이터를 바인딩(연결)시킬 수 있고, ‘ : ‘ 로 축약해서 사용 가능.


src, alt 속성에 userList 요소 객체의 key를 연결시켜준다.


멍멍이 3마리의 프로필 사진이 잘 들어온 것 같다.



최대 3명을 보여줄꺼라 .user_thumb을 3개 만들기는 했는데, userList에 있는 요소 수만큼 알아서 생성되게는 할 수 없을까?

v-for
Vue의 data에 있는 데이터 수만큼 HTML 태그를 반복해주는 속성


v-for를 사용하면 코드를 더 간단하게 표현할 수 있다.

userList의 요소 수만큼 반복하여 .user_thumb을 생성하고, 객체 요소의 데이터들을 바인딩했다. 이렇게 반복해서 생성한 요소들을 나중에 유니크하게 접근해야하는 경우가 생기는데 이럴 때를 대비해 key 속성 값에 각 요소의 index를 연결시켜주는 것이 필요하다. ( :key=”user.index”를 연결하지 않고 실행할 경우 에러가 뜬다.)


멍멍이 4마리가 보인다. v-for를 사용하니 쉽게 배열의 요소 수만큼 알아서 잘 받아오는 것을 확인할 수 있다. 그런데 나는 최대 3개의 썸네일만 보여주기로 했기 때문에 이에 대한 처리를 추가로 해야 한다.


Array 메소드 중에 원하는 부분을 추출할 수 있는 .slice()를 속성에 바로 사용할 수 있는데, 이렇게 하면 좋아요를 누른 사용자 수를 따로 제어할 필요없이 간단하게 구현할 수 있다.


썸네일 3개 이상일 경우 ‘외 N명’ 노출

N명에는 (좋아요를 누른 사람 수) - 3을 해주어야 한다.


HTML에 데이터 연결하기

{{ }}
이중 중괄호로 묶는 탬플릿 표현식. Vue data를 바인딩할 수 있는 텍스트 보간법
반응성이 있어 데이터가 업데이트되는 경우 업데이트된 내용이 반영됨


좋아요를 누른 사람 수는 userList.length와 같다. userList.length를 {{ }}를 사용하여 태그 내 텍스트에 추가하면 된다. 규모가 커질수록 헤깔릴 수 있는 부분이니 totalLikeNum과 같은 변수로 따로 빼는 것도 괜찮을 것 같다.)


값을 잘 받아오고 있습니다. 이제 여기서 3을 빼주면 된다.


만약 썸네일 수도 변수로 빼고 싶을 경우 썸네일 인덱스 값을 데이터 속성으로 등록하고, 속성명을 ‘3’ 자리에 써주면 된다.
