vue에서 당황할 수 있는 순간 - 왜 css가 적용이 안되지?

Vue로 작업하며 만날 수 있는 당황스러운 순간이 있다. 부모 컴포넌트에서 자식 컴포넌트 특정 요소의 css를 수정하고 싶은 순간이 왔다. 그래서 부모 컴포넌트에서 수정을 하였는데.. 적용이 안된다!

이럴 때는 당황하지 말고 ::v-deep 이나 >>>를 사용해보자. 내가 프로젝트에서 사용했을 때에는 자세한 내용을 찾아볼 여유가 없었다.(변명..을 하자면) 빠르게 구글링해서 ‘이게 해답이군’하고 쓰기 바빴다. 오늘은 관련된 정리를 해보고자 한다.


우리가 마주할 수 있는 상황

문제가 되는 상황을 한 줄로 말하면 여러 depth로 선택자 지정하여 추가한 css가 적용이 안될 때 이다.

이러한 상황에 포함될 수 있는 케이스는

  • 부모 컴포넌트에서 자식 컴포넌트의 하위 요소 style 재선언
  • UI 라이브러리 커스텀 (ex. Element UI, materialize, Vuetify..)

왜 이런 일이 발생하는가?

<style>에 scoped 속성을 지정해주었기 때문이다. 선택자에 data-v-블라블라 가 추가되게 하면서 unique해지는 것인데, 이 컴포넌트의 해당 data속성을 가진 선택자에만 적용되도록 해주는 것으로 말 그대로 다른 컴포넌트의 css에 영향을 받지 않고, scoped한 스타일을 유지할 수 있게 된다. 이렇게 되면 부모 컴포넌트에서 지정한 style이 자식 컴포넌트의 하위 요소에는 적용이 안되게 된다.

공식문서를 보면 왜 부모 컴포넌트에서 자식 컴포넌트의 하위 요소에 스타일이 적용이 안되었는지에 대한 어느 정도의 설명이 나온다.

With scoped, the parent component’s styles will not leak into child components. However, a child component’s root node will be affected by both the parent’s scoped CSS and the child’s scoped CSS. This is by design so that the parent can style the child root element for layout purposes.

위의 내용을 정리하면, 부모 컴포넌트에서는 등록한 자식 컴포넌트의 스타일은 적용되지 않는다. 적용되는 유일한 경우는 자식 컴포넌트의 root 요소에만 적용이 되는데 layout 조정용으로 의도된 것이다.


여기서 단순하게 제일 먼저 떠오르는 해결 방법은 scoped 속성을 제거하면 되지 않나?이다.

하지만 이 방법은 별로 추천하지 않는다. 프로젝트 단위가 커질수록 scoped을 설정하여 분리하는 것이 관리도 편하고, 컴포넌트화된 Vue의 사용 목적에도 맞다고 생각된다. (몇몇 컴포넌트에 공통으로 써야하는 스타일이라면 공통 css파일로 분리하여 import 해주는 방식을 지향한다.)

Vue 공식문서에도 Top level인 App과 Layout 관련 컴포넌트는 global로 해도 되지만, 다른 컴포넌트들은 항상 scoped를 설정하는 것을 권장하고 있다.

For applications, styles in a top-level App component and in layout components may be global, but all other components should always be scoped.


어떻게 해결하면 되는가?

적용하고 하는 최종 선택자 앞에 다음 수식어를 붙여주면 된다.

  • /deep/
  • ::v-deep
  • >>>

예시)

/deep/은 현재 deprecated되었으니, ::v-deep나 >>>를 사용하면 되겠다.
만약 SCSS를 사용한다면 >>>는 파싱이 안되니 ::v-deep을 사용해야 한다는 점만 주의하자.

vue-loader Github Issue : /deep/ selector will be deprecated soon


테스트

자식 컴포넌트의 green 글씨를 부모 컴포넌트에서 purple로 바꿔볼 것이다.

style에 scoped 추가시 랜더링 되는 구조

  • 컴포넌트 별 data-v-~ 속성이 추가되어 구분 가능

부모에서 자식의 p태그 color 재선언

  • data 속성으로 지정된 style이 추가될 때 가장 마지막 depth에 추가된다.
  • 부모에서는 자신한테 추가된 자식의 root element에만 접근 가능하고, 자식의 하위 구성 요소들에는 접근을 못한다. 결국 부모 컴포넌트에서 추가한 자식 구성 요소의 스타일이 랜더링이 되지 않는다.

/deep/ or ::v-deep or >>> 추가하기

  • 세 가지 다 테스트 해보았을 때 잘 적용됨
  • 추가할 경우 부모에서 .ex_deep p에 스타일이 적용된 것을 확인 가능.
